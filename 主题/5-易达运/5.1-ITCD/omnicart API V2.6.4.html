接口基础

在v2接口注意事项：

  1. 通过后台获取key和secret

  • key 用于获取token的值

  • secret 用于于校验提交的数据

  1. 一般情况下输入接口的参数为只有3个参数：

  • token 用于确定是否有登录权限，通过api/login接口中获取的登录，有效时间不超过1小时

  • data 传入的数据全部使用

  • sign 用于验证data数据

  1. 增加数据加密字符串

  • data 数据的组成采用 RFC1738 ，经过base64加密
    如要传参数 order_id=10&shipping_id=1
    base64_encode("order_id=10&shipping_id=1");

  1. 数据验证

  • sign 由data的未加密前的数据与secret组成新的字符串再使用md5生成hash.
    使用上面的例子：order_id=10&shipping_id=1
    假如secret的值为#11111#
    md5("#11111#order_id=10&shipping_id=1#11111#")

  1. 返回数据,采用json结构

  • status_no 错误编码.具体查看状态表.

  • msg 提示

  • info 字典

  • list 数组

6.常见字段说明

field name Description
language_id 语言的类型，参考 api/v2/language/lists
customer_field_id 客户自定义字段的id,参考 api/v2/custom_field/lists
order_status_id 订单id，参考 api/v2/order_status/lists
country_id 国家id，参考 api/v2/country/lists
